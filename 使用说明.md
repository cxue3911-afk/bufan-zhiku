# 不凡科技运营助手使用说明

这是你的专属运营 Agent 工作区。之后每次在本项目中打开 Codex 对话，Agent 会优先读取 `AGENTS.md`，并参考各资料夹中的历史内容与你协作。

## 一、日常怎么用

### 1. 写日报
你可以直接说：

```text
帮我写今天日报：
上午做了……
下午做了……
遇到的问题是……
明天计划……
```

Agent 会整理成正式日报，并保存到：

```text
日报/YYYY-MM-DD_日报.md
```

默认日报会采用“时间管理”表格格式，将当天工作拆入 08:30-18:00 的时间段。工作内容和执行方法会尽量分点写，方便查看和汇报。

### 2. 写周报或月报
你可以说：

```text
根据本周日报生成周报。
```

或：

```text
根据本月日报生成月报。
```

Agent 会读取 `日报/` 中的历史文件，汇总重点、成果、问题和下阶段计划。

### 3. 做活动方案
你可以说：

```text
帮我策划一场面向硬科技创业者的线上分享会，主题是 AI for Science 商业化。
```

Agent 会生成完整活动方案，并保存到：

```text
活动方案/YYYY-MM-DD_活动方案.md
```

### 4. 做数据分析
把数据文件放到 `数据报告/` 后，你可以说：

```text
帮我分析最新的数据报告，找出异常、亮点和下一步建议。
```

### 5. 做复盘
你可以说：

```text
根据这次活动数据帮我生成复盘报告。
```

Agent 会保存到：

```text
运营复盘/YYYY-MM-DD_运营复盘.md
```

## 二、资料应该放哪里

- `日报/`：每天的工作记录
- `周报/`：每周总结
- `月报/`：每月总结
- `活动方案/`：活动策划方案
- `内容素材/`：公众号、社群文案、邀请函、访谈提纲
- `数据报告/`：运营数据、活动数据、用户数据
- `用户分层/`：用户画像、分层规则、运营策略
- `运营复盘/`：活动复盘、周期复盘
- `模板库/`：你认可的模板和标准格式
- `参考资料/`：公司介绍、产品资料、领导要求、竞品资料、用户研究

## 三、如何让 Agent 越来越懂你

每当你觉得某个表达、模板、分析方式很好，可以直接说：

```text
这个风格以后沿用。
```

或：

```text
把这个规则记到 AGENTS.md。
```

Agent 会把有价值的经验沉淀到 `AGENTS.md` 的“积累的经验与模板”章节。

## 四、建议你先补充的资料

1. 公司/产品介绍
2. 目标用户画像
3. 近期重点工作
4. 领导常看的汇报格式
5. 已有活动方案或优秀案例
6. 常用数据指标口径
